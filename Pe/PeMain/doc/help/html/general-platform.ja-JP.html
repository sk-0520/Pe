ヘルプ

原則として自身の使用しているプロセッサ幅(32bit/64bit)のPeを使用するのが無難です。

ウィルス検出について

開発中はそうでもなかったのにいざリリースした時に Avast がウィルス検出するようになりました。

  • Windows 7 pro(x64)
  • Avast: virus 140921-0, 2014/09/21 16:09:31
  • Avast: program 2014.9.0.2021

Pe はその性格から 64bit ネイティブで動作さることによる利点は精神衛生以外にはこれと言ってないので32bit版でもいいのですが、だからと言って32bit版ばんじゃーいとは言えない状況です。

ちなみに設定データは 32/64bit で共有されるため再登録する必要はありません。

開発について

コード自体は 32/64bit で共用しているので原理的には大丈夫なんです、原理的には。。。

VSでデザイナ使用時にx86へ切り替える程度でこれと言って固定して開発しているわけではありませんがなんとなーくx86/x64をいい感じ平均的にやっていけたらなーレベルで切り替えてます。

使用について

冒頭に記述している通り、自身の使用環境に合わせるのが最適なんですが諸々の事情で64bitCPUでも32bitプログラムを使用しないといけないかもしれないので、作者としてはどっちでもお好きにどうぞとしか言いようがありません。

Windows から提供される環境変数 PROGRAMFILES と PROGRAMFILES(X86) が 32/64bit でそれぞれ扱いが変わってきたりするので自動登録などでは登録できるのに使用できないことがあります。
